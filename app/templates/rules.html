How rules work

  
    
    
  • Threshold fires when the metric crosses a value (edge-triggered — once per crossing, not every poll). Optional hysteresis stops it flapping near the boundary; fire on clear also fires when it drops back.
    • 
    
  • On change fires whenever the value moves (optionally ignore moves smaller than min change).
    • 
    
  • Every tick fires on every poll — use this for continuous streaming to Spot.
    • 
    
  • Cooldown: after firing, wait this many seconds before firing again.
    • 
    
  • Templates accept {value}, {value:.1f}, {metric_key}, {source_name}, {unit}, {observed_at}, {now}.
    • 
    
  • Aggregated rules: pick ★ Current weather (aggregated) as the source to bind a rule to the deduped readout (weather.wgbt, …). The rule follows whichever source currently wins that field and fails over automatically if that feed dies.
    • 
    
  • Stale-data guard: every source maintains <slug>._data_age_seconds — seconds since the feed last produced fresh data. It keeps counting up if the endpoint dies or freezes, so a rule like _data_age_seconds > 600 alerts on a dead feed.
    • 
    
  • Lightning: lightningDelay is the hold countdown in seconds — a strike (re)starts the 10-minute hold at 600 and it counts down; 0 means all clear. Use > 0 with fire on clear to catch both the hold start and the all-clear (mid-hold resets to 600 won't re-fire).
